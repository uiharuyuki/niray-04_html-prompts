# 05_画像アセット策定

旧 `05-1_画像提案.md` / `05-2_画像アセット表の確定.md` / `05-3_画像アセット表.md化.md` を統合した工程です。
画像の **提案 → 採用判断 → `image-assets.md` としての確定出力** を、1つの prompt で進めます。

ChatGPTでの対話を想定しています。最初に Part A を投げ、回答を見て採用判断を返したあと、Part B を投げて `image-assets.md` を出力させてください。

---

## Part A: 画像案の提案

```text
`project.md` とスライド構成・原稿・設計案をもとに、
画像付きHTMLスライドで使用する画像案を提案してください。

目的:
最終的に `image-assets.md` を作成するための、画像候補・役割・方向性を整理します。
このターンでは、まだ採用確定や `image-assets.md` 形式での出力はしません。

重要方針:
- すべてのスライドに無理に画像を入れない
- 画像は装飾ではなく、理解・印象・説得力を強めるために使う
- 画像内には、短いタイトル、コピー、ラベル、UI文言を必要に応じて入れてよい
- 長文説明、細かい注釈、発表時の補足、あとから変更されやすい文章はHTML側に残す
- 普通のPowerPoint風にせず、提案資料として使える範囲でポスター、雑誌表紙、editorial proposal graphic のような主張を持たせる
- 表紙は強め、UI説明は説明性を残して少し華やか、導線説明は分かりやすく平板にしない

やってほしいこと:
1. 画像が必要なスライドと、画像なしでよいスライドを分ける
2. 各画像の役割を整理する
3. 画像ごとの構図、比率、トーンを提案する
4. 画像ごとにデザイン温度（強め / 中程度 / 説明寄り）を設定する
5. 表紙、UI説明、導線説明、世界観画像、比較画像など、画像タイプを分ける
6. 人物・キャラクターを使うべき画像と、使わないほうがよい画像を分ける
7. 画像内に入れる文字と、HTML側に残す文字を分ける
8. 似た構図が続かないように、画像ごとの見せ方に差をつける
9. 後でリネームしやすいように、仮ファイル名を提案する

画像アセット表の項目（このターンでの一次提案として埋める）:
- 仮ファイル名
- 使用スライド
- 画像の役割
- 画像タイプ
- デザイン温度
- 見た目の方向性
- 主役要素
- 推奨比率
- 構図
- 人物・キャラクターの有無
- 画像内文字の有無
- 画像内に入れる文言案
- HTML側に残す文言
- HTML上での配置
- トーン
- 注意点

出力条件:
- 画像を増やしすぎない
- 1画像1役割を基本にする
- 同じような構図を連続させない
- キャラクターや人物を無理に全画像へ入れない
- 画像内文字は短く読みやすい文言に限定する
- 画像内文字は、必ず具体的な文言まで提案する
- HTML側で文字を載せる前提の画像では、余白を確保する
- まだ画像生成promptは作らない
- まだ `image-assets.md` 形式では出力しない（次のPart Bで行う）

入力情報:
---
【前提資料】
`project.md` とスライド構成・原稿・設計案を貼ります。

【補足・変更点】
-

【画像について迷っていること】
-

【使いたいキャラクター・参考画像】
-

【デザインの方向性】
-

【避けたい表現】
-
---
```

---

## Part B: 採用判断 + `image-assets.md` 出力

Part A の提案に対して、採用・修正・不採用・追加の方針を入力欄に書き、Part B を投げます。
このターンの出力は、そのまま `image-assets.md` として保存できる形式（ダウンロード優先、不可時はコードブロック）にします。

```text
Part A で提案した画像案と、以下の採用判断をもとに、
HTML化工程に渡す `image-assets.md` を確定版として出力してください。

目的:
この `image-assets.md` は、画像付きHTMLスライドを作成する際の、
画像配置・画像役割・ファイル名管理・画像内文字・デザイン温度の正本として使います。

「どの画像を、どのsectionで、どの役割として、どの文字重複に注意して使うか」が分かる、
画像仕様書として整理してください。

判断方針:
- 必要以上に新しい案を増やさない
- 採用・修正して採用・不採用・追加 を明示する
- 画像内文字とHTML側テキストが重複しすぎないようにする
- 表紙は強め、UI説明は説明性を残して少し華やか、導線説明は分かりやすく平板にしない
- 未確定の情報は推測で確定扱いせず「要確認」と書く

入力情報:
---
【採用する画像案】
-

【修正して採用したい画像案】
-

【採用しない画像案】
-

【追加したい画像案】
-

【人物・キャラクターの扱い】
-

【画像内文字の扱い】
-

【画像内に入れたい文言の最終方針】
-

【HTML側に残したい文言の最終方針】
-

【デザインの最終方針】
-

【ファイル名ルール】
例: 01_cover.png / 02_flow.png / 03_ui.png のように、番号＋用途が分かる名前にする

【HTML化時に特に守りたいこと】
-

【判断してほしいこと】
-
---

出力フォーマット:
出力は `image-assets.md` としてそのまま保存できる形式にしてください。
ダウンロードできるファイル形式（タップ／クリックで保存できる形）を第一とし、出せない場合のみフォールバックとして1つのMarkdownコードブロックで出力してください。
ファイル名は `image-assets.md` とし、`-main` などの接尾辞・サフィックスを付けないでください。
ファイルやコードブロックの外には前置きや補足を書かないでください。

以下の構成で出力してください。

## 1. 概要
- この資料の目的
- 画像の使い方の基本方針
- 画像内文字の基本方針
- デザイン温度の基本方針
- HTML化時に守ること
- Claude Code / Codex に優先して読ませるべき内容

## 2. 画像一覧表
以下の項目で表に整理してください。

- リネーム後のファイル名
- 使用セクション
- 画像の役割
- 画像タイプ
- デザイン温度
- 見た目の方向性
- 主役要素
- 推奨比率
- 画像内文字の有無
- 画像内に入れる文言
- HTML側に残す文言
- 人物・キャラクターの有無
- HTML上での配置方針
- alt属性の案
- 注意点

## 3. 各画像の詳細仕様
画像ごとに以下を整理してください。

- ファイル名
- 使用するスライド / section
- この画像の目的
- 画像タイプ
- デザイン温度
- 見た目の方向性
- 主役要素
- 画像で見せるべき要素
- HTML側で補足するべき要素
- 画像内に入れる正確な文言
- 画像内文字の役割
- HTML側で重複させない文言
- HTML側に残す文言
- 推奨レイアウト
- object-fit / object-position の注意
- 人物・キャラクターの扱い
- 避けたい使い方
- 画像内文字の検品ポイント
- alt属性案

## 4. 画像内文字の共通ルール
- 画像内文字は短く、大きく、読みやすくする
- 画像内文字は日本語を基本にする
- 指定していない謎の文字や意味不明な英字を入れない
- 長文説明や細かい注釈はHTML側に残す
- 画像内文字とHTML側見出しを重複させすぎない
- 画像内文字が読みにくい場合は、HTML側に補助テキストを置く
- 画像内文字に誤字や崩れがある場合、画像を編集せず注意点として報告する
- alt属性には、画像内文字が意味を持つ場合、その内容も含める

## 5. HTML化時の共通ルール
- 画像の縦横比を崩さない
- 画像を装飾だけで使わない
- 1画像1役割を基本にする
- スマホ幅では画像が潰れないようにする
- 重要部分が見切れないようにする
- 画像上にHTMLテキストを重ねる場合は、必ず可読性を確保する
- 画像内にメインタイトルがある場合、HTML側の同じタイトルは控えめにするか補足的に扱う

## 6. Claude Code / Codex への引き継ぎメモ
- 画像配置はこの `image-assets.md` を優先すること
- `project.md` と矛盾する場合、画像配置については `image-assets.md` を優先すること
- 実際の画像ファイル名を確認して参照すること
- 画像ファイルが存在しない場合は、勝手に別画像で代用せず報告すること
- 二重拡張子や指定と違うファイル名がある場合は報告すること
- 画像ファイル自体は編集しないこと
- 画像内文字が読みにくい場合は、画像を編集せず、HTML側の補助テキストや配置で補うこと

出力条件:
- 表は読みやすく整理する
- 未確定の情報は推測で確定扱いせず「要確認」と書く
- 画像内に入れる文言は具体的に書く
- HTML側に残す文言も明記する
- デザイン温度と見た目の方向性を必ず各画像に設定する
- 次工程の画像生成prompt作成（06）で必ず守る条件を、画像ごとの「注意点」に反映する
```

---

## 完了条件

- 画像案が採用・修正・不採用・追加に分かれている
- 画像ごとに役割・配置・画像内文字・HTML側文言・デザイン温度が決まっている
- そのまま保存できる `image-assets.md`（ダウンロード形式、不可時はMarkdownコードブロック）が出力されている
- 06（画像生成prompt作成）で必要な条件が、画像ごとの注意点に反映されている
