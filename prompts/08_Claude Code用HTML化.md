# 08_Claude Code用HTML化

```text
このフォルダ内の資料と画像をもとに、画像付きHTMLスライド資料を新規作成してください。

## 目的

フォルダ内の `project.md`、`image-assets.md`、画像素材、必要に応じて `implementation-brief.md` を使い、
発表・提案・説明に使えるHTMLスライド資料を作成してください。

単なる縦長ページではなく、
`section` 単位で見せ場が分かれる画像付きHTMLスライド資料として構成してください。

## 正本

- 企画内容・文章・構成・トーン: `project.md`
- 画像の使用場所・役割・ファイル名・配置方針・画像内文字: `image-assets.md`
- 実装許可範囲・禁止事項・検証条件: `implementation-brief.md`

`project.md` と `image-assets.md` が矛盾する場合、
企画内容は `project.md`、画像配置と画像内文字の扱いは `image-assets.md` を優先してください。

## 作成してほしいファイル

- `index.html`
- `style.css`
- `script.js`

`script.js` は必要最小限で構いません。
JavaScriptなしでも最低限内容が読めるHTML構造にしてください。

## 編集しないもの

- `project.md`
- `image-assets.md`
- `implementation-brief.md`
- 画像ファイル

不足している画像がある場合は、勝手に別画像で代用せず報告してください。
画像内文字に誤字・崩れ・読みにくさがある場合も、画像を編集せず注意点として報告してください。

## 実装条件

- HTML / CSS / JavaScript のみで作成
- 外部ライブラリ、CDN、外部フォントは使わない
- `file:///` で直接開いても表示できる
- 画像はローカル相対パスで参照する
- コンソールエラーが出ないようにする
- CSSは後から修正しやすい構成にする
- 画像の縦横比を崩さない
- 画像内文字がある前提で、HTML側のテキスト量と配置を調整する

## 構成方針

- 1 section = 1スライドを基本にする
- `project.md` の文章をそのまま長文で貼りすぎない
- 画面に載せる文章は短く整理する
- 表紙、概要、課題、提案、導線、UI、比較、まとめなど、内容に応じて自然な流れにする
- 最後は締めのメッセージが伝わるまとめsectionにする
- 情報量が多い場合は、カード、比較表、注記、図解風レイアウトに分けて整理する
- 画像内に大きなタイトルやコピーがあるsectionでは、HTML側の見出しを控えめにする

## 画像配置方針

- `image-assets.md` に従って、各画像を適切なsectionに配置する
- 画像をただ並べず、sectionのメッセージを補強する形で使う
- 画像内タイトルとHTML大見出しを同じ強さで二重表示しない
- 画像内文字をHTML要素で隠さない
- 重要部分が見切れないように `object-fit` / `object-position` を調整する
- 画像上にHTMLテキストを重ねる場合は可読性を確保する
- 画像内文字がスマホ幅で小さくなる場合は、画像下に補助テキストを置く
- すべての画像に適切な `alt` 属性を入れる
- 画像内文字が意味を持つ場合、`alt` 属性にもその内容を含める

## デザイン方針

- 画像を大きく使い、見た目の印象を強める
- 普通のPowerPointのように硬すぎる見た目にはしない
- ただし、提案資料として読みにくいほど派手にはしない
- 余白、見出し、カード、画像のバランスを整える
- 文字サイズは発表時に読みやすい大きさにする
- 色数を増やしすぎず、資料全体の統一感を優先する
- PC発表を主目的にしつつ、スマホ幅でも大きく崩れないようにする
- `image-assets.md` に記載されたデザイン温度を反映する

## デザイン温度の扱い

`image-assets.md` のデザイン温度を、HTML側のレイアウトにも反映してください。

### 強め（表紙・キービジュアル向け）

- 画像を主役として大きく扱う
- HTML側の見出しを画像に重ねすぎない
- 画像内タイトルが強い場合、HTML側は資料名や短い補足に留める
- 余白を使い、ポスターや雑誌表紙のような印象を壊さない
- 画像の周囲に過剰なカードや説明を置かない

### 中程度（導線説明・世界観・比較画像向け）

- 画像と説明文のバランスを取る
- 画像を大きく見せつつ、HTML側で要点を補足する
- 図解・ラベル・カード・本文が競合しないように整理する
- 画像内文字とHTML側の説明が重複しすぎないようにする

### 説明寄り（UI説明画像向け）

- UIやスマホ画面を見やすく配置し、小さくしすぎない
- HTML側では、UIで何ができるのかを短く補足する
- 画像内のUIラベルとHTML側の説明が二重にならないようにする
- スマホ幅では画像を上、説明を下にするなど読みやすさを優先する

## section別の扱い

### 表紙section

- 表紙画像に大きなタイトルやコピーがある場合は、画像を主役にする
- 画像内タイトルを活かし、HTML側で同じ大見出しを画像の上に重ねない
- 必要なら、画像の外側または下部に小さく資料名や補足を置く
- スマホでは画像を16:9のまま縮小し、必要なら画像下に補助見出しを追加する

### UI説明section

- スマホUIや画面モックを大きく見せる
- 画像が小さくなりすぎる場合は、左右分割より上下構成を優先する
- 画像内にUIラベルやボタン文言がある場合、HTML側で同じ文言を繰り返さない
- 「提案用デモ」「本採用後に実装予定」など、画像だけでは誤解されやすい補足をHTML側に置く

### 導線説明section

- 証明書・QR・LP・店舗・地図などの関係を説明する画像として扱う
- 画像内ラベルがある場合は活かし、HTML側本文を短くする
- 画像内の流れとHTML側の箇条書きが重複しすぎないようにする
- スマホ幅では画像が潰れないように縦積み構成にする

### 世界観画像・比較画像

- 映画案・町並み案などの画像は大きく見せる
- 画像内に案名やコピーがある場合、HTML側の補足説明は短くする
- 画像の上に説明カードを重ねる場合は、スマホで可読性が崩れないようにする
- スマホでは画像と説明カードを分離して縦に並べてもよい
- 画像内文字を覆い隠す位置にHTML要素を重ねない

## レスポンシブ確認幅

以下の幅を想定して調整してください。

- 1920px前後
- 1366px前後
- 768px前後
- 390px前後

確認ポイント:
- 横スクロールが出ない
- スマホでは基本1カラムで読める
- 画像が潰れない
- 画像内文字が小さくなりすぎない
- 文字が小さすぎない
- カードや比較表がスマホでも読める
- 固定ナビがある場合、スマホで邪魔にならない
- 画像とHTMLテキストの重なりがスマホで崩れない

## 作業手順

1. フォルダ内のファイル一覧を確認する
2. `project.md` を読む
3. `image-assets.md` を読む
4. `implementation-brief.md` があれば読む
5. 使用する画像ファイルを確認する
6. 画像ファイル名に二重拡張子や指定違いがないか確認する
7. HTMLスライド全体の構成を決める
8. `index.html` / `style.css` / `script.js` を新規作成する
9. 画像パス、alt属性、レスポンシブ、表示崩れを確認する
10. 画像内文字とHTML側テキストが重複しすぎていないか確認する

## 最終確認

作成後、以下を確認してください。

1. `index.html` を `file:///` で開ける構成になっている
2. 画像パスが正しい
3. `image-assets.md` どおりに画像が使われている
4. 画像内文字方針が反映されている
5. デザイン温度がsection設計に反映されている
6. `project.md` の内容と成果物の内容が大きくズレていない
7. 画像内文字とHTML側見出しが重複しすぎていない
8. スマホ幅で横スクロールが出ていない
9. コンソールエラーが出ていない
10. 最後のまとめsectionが締めとして成立している

## 完了後の報告

以下を簡潔に報告してください。

- 作成したファイル
- 主な実装内容
- 使用した画像と配置したsection
- `image-assets.md` から反映した内容
- 画像内文字とHTML側テキストの重複を避けるために行ったこと
- レスポンシブ対応で行ったこと
- 確認した表示条件
- 残っている注意点
```
