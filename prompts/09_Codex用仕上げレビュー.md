# 09_Codex用仕上げレビュー

```text
このフォルダ内のHTMLスライド資料を、仕上げレビューして必要な修正を行ってください。

## 目的

Claude Codeなどで作成した画像付きHTMLスライド資料を、
最終提出・発表に使える品質まで整えることです。

大幅な作り直しではなく、既存の構成・デザイン方針を活かしながら、
表示崩れ、可読性、画像配置、画像内文字、レスポンシブ、細部の完成度を改善してください。

## 作業前提

- `index.html` / `style.css` / `script.js` を確認してください
- `project.md` がある場合は、企画内容・構成・トーンの正本として確認してください
- `image-assets.md` がある場合は、画像の使用場所・役割・配置方針・画像内文字・デザイン温度の正本として確認してください
- `implementation-brief.md` がある場合は、実装許可範囲・禁止事項・完了条件として確認してください
- スクリーンショットがある場合は参考にしつつ、実ファイルのコードと表示状態を正として確認してください
- 画像ファイル自体は編集しないでください
- 外部ライブラリ、CDN、外部フォントは追加しないでください

## 編集してよいファイル

- `index.html`
- `style.css`
- `script.js`

## 編集しないファイル

- `project.md`
- `image-assets.md`
- `implementation-brief.md`
- 画像ファイル

画像内文字に誤字・崩れ・読みにくさがある場合は、画像を編集せず、注意点として報告してください。

## 優先順位

1. 表示崩れ、横スクロール、画像パスミス、コンソールエラー
2. スマホ表示の崩れ
3. 画像内文字とHTML側テキストの重複・干渉
4. 画像内文字の読みにくさ
5. 文字の可読性
6. 画像の見切れ・配置ミス
7. 余白・カード・装飾の細部調整
8. PowerPointっぽく硬すぎる見た目の軽微な改善
9. 文章量の軽微な整理

## 確認してほしい観点

### 1. 全体構成
- sectionの順番が自然か
- 1 section = 1メッセージになっているか
- `project.md` の内容とHTMLの内容が大きくズレていないか
- 情報量が多すぎるsectionがないか
- 最後のまとめsectionが締めとして成立しているか

### 2. デザイン・見た目
- PowerPointっぽく硬すぎないか
- 画像付きHTMLスライド資料として見栄えがあるか
- 表紙画像の強さをHTML側のレイアウトが弱めていないか
- UI説明sectionが地味な図解になりすぎていないか
- 余白が広すぎる、または詰まりすぎていないか
- 見出し、本文、カード、画像の優先順位が分かりやすいか
- 色、角丸、影、線、背景の使い方に統一感があるか

### 3. 画像配置
- `image-assets.md` の指定どおりに画像が使われているか
- 画像がただ置かれているだけになっていないか
- sectionのメッセージを補強する配置になっているか
- 画像の縦横比が崩れていないか
- 重要部分が見切れていないか
- `object-fit` / `object-position` が適切か
- 画像上にHTML要素を重ねている場合、画像内文字を隠していないか
- すべての画像に適切な `alt` 属性があるか

### 4. 画像内文字とHTML側テキスト
- 画像内タイトルとHTML側見出しが重複しすぎていないか
- 画像内コピーとHTML側リード文が同じ内容を繰り返していないか
- 画像内文字の上にHTMLテキストやカードが重なっていないか
- 画像内文字がスマホ幅で読めるか
- 画像内文字が小さくなる場合、HTML側に補助テキストがあるか
- 画像内文字に誤字・崩れ・不要な文字がないか

### 5. 可読性
- 見出しが読みやすいか
- 本文が長すぎないか
- 文字サイズが小さすぎないか
- 行間、字間、段落間が読みやすいか
- カード内テキストや注記が小さすぎないか
- スマホ幅で文字が詰まっていないか

### 6. レスポンシブ
以下の幅を想定して確認・修正してください。

- 1920px前後
- 1366px前後
- 768px前後
- 390px前後

確認ポイント:
- 横スクロールが出ない
- スマホでは基本1カラムで読める
- 画像が潰れない
- 画像内文字が小さくなりすぎない
- カードや比較表がスマホでも読める
- 固定ナビや装飾要素がスマホで邪魔にならない
- 画像とHTMLテキストの重なりがスマホで破綻しない

### 7. 実装品質
- HTML構造が破綻していないか
- 見出し階層が自然か
- CSSが過剰に複雑になっていないか
- 不要な重複CSSがないか
- JavaScriptにエラーがないか
- JavaScriptなしでも最低限内容が読めるか
- `file:///` で開いても画像やCSS/JSが読み込まれるか
- 画像パスが実ファイル名と一致しているか
- 二重拡張子や指定違いがないか

## 修正方針

- 大幅なデザイン変更はしない
- 既存の良い部分は残す
- 可読性、レスポンシブ、画像配置、画像内文字の見え方、余白の問題は積極的に直す
- `project.md` と `image-assets.md` の意図から外れる修正は避ける
- 表紙の強いビジュアルを、HTML側の重複見出しや過剰な装飾で弱めない
- UI説明は、説明性を落とさずに視覚的な見やすさを調整する
- 画像ファイル自体は編集しない

## 作業手順

1. フォルダ内のファイル一覧を確認する
2. `project.md` と `image-assets.md` を読む
3. `implementation-brief.md` があれば読む
4. `index.html` / `style.css` / `script.js` を確認する
5. 実ファイルをもとに表示崩れや実装上の問題を確認する
6. 画像内文字とHTML側テキストの重複を確認する
7. 必要な修正を `index.html` / `style.css` / `script.js` に反映する
8. 可能であれば `file:///` またはローカルサーバーで表示確認する
9. PC幅、タブレット幅、スマホ幅を想定してCSSを調整する

## 完了後の報告

以下を簡潔に報告してください。

- 修正したファイル
- 主な修正内容
- `project.md` / `image-assets.md` と照合して直した点
- 画像内文字とHTML側テキストの重複を避けるために直した点
- レスポンシブ対応で直した点
- 画像配置で直した点
- 画像内文字に誤字・崩れ・読みにくさがある場合の報告
- 残っている注意点
- 追加で確認したほうがよい表示幅や箇所
```
