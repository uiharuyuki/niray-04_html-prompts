# 06-1_画像一括生成（Codex スキル `/imggen`）

`06`（ChatGPT で1枚ずつ作る丁寧ルート）の代替。**Codex が `image-assets.md` の画像を一括生成して停止**する、後半の入口です。
作る人＝直す人なので、ファイル名・比率の事故が起きにくく、修正もその場で速い。

> このファイルを `SKILL.md` として Codex の skills ディレクトリに置くと、作業フォルダで `/imggen`（または「画像を作って」）で発動します。`name: imggen` がスラッシュコマンド名。frontmatter は消さないこと。スキル登録していない場合は、本文をそのまま貼っても動きます（自己完結）。

## 目的

`project.md` と `image-assets.md` を読み、`image-assets.md` に書かれた画像を**すべて生成**してカレントフォルダ直下に保存する。**実装（HTML/CSS/JS）はやらない。**生成後は停止し、人間が画像を目で見る。

## 正本として読むファイル

1. `project.md` … 企画・構成・トーン
2. `image-assets.md` … 画像のファイル名・比率・役割・画像内文字の発注書（**ファイル名と比率はこれに厳密に従う**）

## 進め方

0. **カレントフォルダの中身を自分で確認する**。`project.md` / `image-assets.md` を読み、既にある画像（キャラクターシート等）は参考画像として扱う。パスは聞かない。
1. `image-assets.md` の順に、**まだ無い画像だけ**生成する。「画像なし」スライド、「既存流用・再生成不要」と書かれた画像は**生成しない**。
2. 生成画像を `image-assets.md` のファイル名で**カレントフォルダ直下**に保存する（例: `01_cover.png`）。
3. **停止して報告する**：生成した画像一覧、流用した既存画像、生成しなかったもの。「画像を確認してください。問題なければ 08（Claude Code）で実装へ。直したい場合はこのまま指示してください」と伝える。**実装には進まない。**

## 画像生成の方針（おしゃれさの肝）

- **枚数は `image-assets.md` に従う**。生成は時間・usage が大きいので勝手に増やさない。
- **参考画像（キャラクターシート等）は同一性の参考のみ**＝顔・髪・服装だけ踏襲。**ポーズ・構図・背景は新規に作る。棒立ちポーズをそのまま再現しない。** キャラは必ず“シーンの中で何かしている”状態にする。
- **フラットな単色／虹グラデ背景にキャラを貼るのは禁止**。奥行きのあるシーン・環境を作る。配色は**2〜3色＋グラデ方向1つ**に統一。
- **レイアウトのタイトル・見出しは画像に焼き込まない**（HTML側）。画像内文字はロゴ的な短い刻印のみ、すべて日本語。指定外の文字・意味不明な文字・不要な英字・架空のロゴ文字は入れない。
- スライドごとに構図を変える。表紙は雑誌表紙・ポスター・キャンペーンキービジュアルのように強め。
- ネガティブ指定: standard PowerPoint look / safe corporate design / weak hierarchy / flat gradient background with pasted character / character-sheet standing pose / baked-in layout text / random English text / meaningless pseudo text / unnecessary logos / rainbow clutter。

## 修正依頼の扱い（この工程の運用）

生成画像を見て直したいときは、**直しの種類で対応を分ける**。

### 通常修正（その場で対応する）

文字直し・キャラのサイズ調整・余白・色味・構図の微調整など。

- **このセッションでそのまま壁打ち＋再生成する**。発注書に投げ直さなくてよい。
- **画像内文字・画像の役割・alt を変えた場合は、`image-assets.md` も同期更新する**（下の「`image-assets.md` 編集の線引き」に従う）。
- 構図・色・サイズだけの微調整で、発注内容（文字・役割）が変わらないなら、`image-assets.md` は触らなくてよい。

### 全体方向性の見直し（Codex で深追いしない）

「PowerPointっぽい」「表紙が弱い」「構図から作り直したい」など、方向性レベルの作り直し。

- **Codex でこね続けない。** 方向性の視覚判断は ChatGPT のチャット（画像を見ながらの壁打ち）の方が強い。
- 流れ：①ChatGPT で画像を見ながら壁打ち → ②フル版 `07_画像改善prompt作成.md`（全体見直し系）で改善方針・再生成promptを固める → ③固めた方針をこの Codex に渡して再生成。
- Codex は「方針を実行して再生成する側」に回す。

## `image-assets.md` 編集の線引き（重要）

`image-assets.md` は本来「編集しない」正本だが、**この 06-1 工程に限り Codex が編集してよい**（Codex が画像の作者なので、発注書を同じ人が直すのが一番ズレない）。ただし条件を守る。

- **触るのは `image-assets.md` だけ。`project.md` は触らない**（企画は人間／ChatGPT の領域。画像修正が企画に波及したら人間に報告する）。
- **変わった項目だけ外科的に直す**（画像内文字・役割・alt・注意点）。表構成を作り直さない。
- **何をどう変えたか差分を報告する**（どの画像の何を直したか）。
- **画像の実体と発注書を必ず一致させてから止まる**（次工程の 08 が古い発注書を読まないように）。
- **08（Claude Code 実装）・09（Codex レビュー）では `image-assets.md` は凍結（編集禁止）**。CLAUDE.md / `00_共通ルール.md` の「image-assets.md を編集しない」ルールは、この 08／09 側に適用される。

## 完了条件

- `image-assets.md` の生成対象画像がすべて生成・保存され、ファイル名が発注書と一致している。
- 修正でファイル内文字・役割を変えた場合、`image-assets.md` が画像の実体と一致している。
- **停止して報告済み。実装には進んでいない。**

## 再生成・部分修正のコマンド例

- 特定の画像だけ作り直す：`/imggen 表紙だけ作り直して`
- 文字だけ直す：`/imggen 03_ui の画像内文字を「予約する」に変えて、image-assets.md も直して`
- 方向性から見直す：いったん停止し、ChatGPT 壁打ち→07 で方針を固めてから渡す。

スキル登録済みなら作業フォルダで `/imggen` と打つだけ。読むファイル・参考画像・出力先はすべてカレントフォルダ。例外を伝えたいときだけ一言添える。
