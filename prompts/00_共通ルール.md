# 00_共通ルール

このファイルは、各工程で共通して守るルールです。
個別プロンプトが長くなりすぎる場合は、この共通ルールを前提として渡してください。

## 基本方針

- 最終成果物は、画像付きのHTMLスライド資料
- 単なる縦長ページではなく、`section` 単位で見せ場が分かれる資料にする
- 1 section = 1メッセージを基本にする
- 不明点だけで止まらず、仮定を置ける範囲は進める
- ただし、方針や成果物品質に大きく関わる不明点は「確認が必要」と明記する
- 推測で確定事項を作らない

## 正本の扱い

実装・レビュー時の優先順位は以下です。

1. `project.md`: 企画内容、構成、文章、トーンの正本
2. `image-assets.md`: 画像の使用場所、役割、配置方針、画像内文字、alt属性の正本
3. `implementation-brief.md`: 実装許可範囲、禁止事項、検証条件
4. 実ファイル: `index.html`、`style.css`、`script.js`、画像ファイル

`project.md` と `image-assets.md` が矛盾する場合は、企画内容は `project.md`、画像配置と画像内文字の扱いは `image-assets.md` を優先します。

## .md 成果物の出力ルール

`project.md` / `image-assets.md` など、このワークフローの正本となる `.md` をAIに出力させるときは、以下を守ります。

- 出力は **ダウンロードできるファイル形式**（タップ／クリックで保存できる形）を第一にする
- ダウンロード形式で出せない環境・モードの場合のみ、フォールバックとして **1つのMarkdownコードブロック** で出力する
- どちらの場合も、ファイルやコードブロックの外に前置き・補足を書かない
- ファイル名は指定どおり正確に付ける（例: `project.md` / `image-assets.md`）。`-main` などの接尾辞・サフィックスを勝手に付けない
  - ※ ダウンロード時のファイル名はAI側のツール挙動で接尾辞が付くことがある。付いた場合は保存後に正しい名前へ直す

## 編集してよいもの・いけないもの

実装・レビュー工程では、基本的に以下を守ります。

編集してよい:
- `index.html`
- `style.css`
- `script.js`

編集しない:
- `project.md`
- `image-assets.md`（ただし **06-1（画像一括生成）工程に限り Codex が編集可**。下記の例外を参照）
- 画像ファイル
- 元のプロンプトファイル

`image-assets.md` 編集の例外（06-1工程のみ）:
- 06-1 で生成画像を直し、**画像内文字・役割・alt を変えた場合は、Codex が `image-assets.md` も同期更新する**（作る人＝直す人で発注書のズレを防ぐため）。
- 触るのは `image-assets.md` だけ。`project.md` は触らない。
- 変わった項目だけ外科的に直し、表構成は作り直さない。差分を報告する。
- **08（HTML化）・09（レビュー）では `image-assets.md` は凍結（編集禁止）**。上の「編集しない」はこの 08／09 側に適用される。

画像内文字に誤字・崩れ・読みにくさがある場合は、画像を編集せず、注意点として報告します。

## 画像内文字のルール

画像内に入れてよい:
- メインタイトル
- 短いコピー
- 案名
- UIラベル
- ボタン文言
- 図解ラベル
- 短い吹き出し
- 地図や店舗カードの短いラベル

HTML側に残す:
- 長文説明
- 箇条書き本文
- 条件説明
- 細かい注釈
- 発表時の補足
- あとから修正されそうな文章
- 誤字が許されない正式文言

注意:
- 画像内タイトルとHTML大見出しを同じ強さで二重表示しない
- 画像内文字をHTMLカードや見出しで隠さない
- スマホ幅で画像内文字が読みにくい場合は、HTML側に補助テキストを置く
- 画像内文字が意味を持つ場合は、alt属性にも含める

## デザイン方針

- 普通のPowerPoint風に寄せすぎない
- ポスター、雑誌表紙、雑誌特集扉、editorial proposal graphic のような主張を持たせる
- ただし、提案資料として読みにくいほど派手にしない
- 画像、文字、余白を分離させず、1つの画面として整理する
- 見出し、本文、カード、画像の優先順位を明確にする
- 色数を増やしすぎず、資料全体の統一感を優先する

## 実装条件

- HTML / CSS / JavaScript のみ
- 外部ライブラリ、CDN、外部フォントは使わない
- `file:///` で直接開いても表示できる
- 画像はローカル相対パスで参照する
- JavaScriptなしでも最低限内容が読めるHTML構造にする
- 画像の縦横比を崩さない
- 横スクロールを出さない

## 検証幅

可能であれば以下の幅で確認します。

- 1920px前後
- 1366px前後
- 768px前後
- 390px前後

確認項目:
- 横スクロールが出ない
- 画像パスが正しい
- コンソールエラーがない
- スマホでは基本1カラムで読める
- 画像が潰れない
- 画像内文字が小さくなりすぎない
- 文字と画像が重なって読みにくくならない
