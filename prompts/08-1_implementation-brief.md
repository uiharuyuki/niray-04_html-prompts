# 08-1_implementation-brief

このファイルは、Claude Code / Codex にHTML化を依頼する直前に渡す短い実装ブリーフです。
`project.md` と `image-assets.md` が長い場合でも、実装上の許可範囲と完了条件を明確にできます。

```text
# implementation-brief

## 今回作るもの

画像付きHTMLスライド資料を作成してください。
単なる縦長ページではなく、1 section = 1スライドとして見せ場が分かれる構成にしてください。

## 正本として読むファイル

1. `project.md`
   - 企画内容、構成、文章、トーンの正本
2. `image-assets.md`
   - 画像の使用場所、役割、配置、画像内文字、alt属性の正本
3. この `implementation-brief.md`
   - 実装許可範囲、禁止事項、完了条件

## 編集してよいファイル

- `index.html`
- `style.css`
- `script.js`

## 編集してはいけないファイル

- `project.md`
- `image-assets.md`
- 画像ファイル
- 元のプロンプトファイル

## 必須条件

- HTML / CSS / JavaScript のみ
- 外部ライブラリ、CDN、外部フォントなし
- `file:///` で直接開ける
- 画像はローカル相対パスで参照する
- 画像ファイル名は実ファイルを確認して使う
- 画像の縦横比を崩さない
- 横スクロールを出さない
- コンソールエラーを出さない
- JavaScriptなしでも最低限内容が読める
- 画像内文字とHTML側見出しを重複させすぎない
- 画像内文字がスマホで読みにくい場合はHTML側に補助テキストを置く

## 画像ファイルの扱い

- 画像ファイル自体は編集しない
- 不足画像がある場合は、勝手に別画像で代用せず報告する
- 二重拡張子や指定と違うファイル名がある場合は、実ファイル名を確認して正しく参照し、報告する
- 画像内文字に誤字・崩れ・読みにくさがある場合は、画像を編集せず注意点として報告する

## 確認する表示幅

- 1366px前後
- 768px前後
- 390px前後

可能であれば 1920px前後も確認してください。

## 完了条件

- `index.html`、`style.css`、必要なら `script.js` が作成されている
- `file:///` で表示できる構成になっている
- 画像パスが正しい
- 横スクロールがない
- スマホ幅で基本1カラムで読める
- 画像が潰れていない
- コンソールエラーがない
- `project.md` と `image-assets.md` の意図から大きく外れていない
- 残課題があれば完了報告で明記されている
```
