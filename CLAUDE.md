# Project Instructions

This file is intentionally synchronized with the counterpart agent instruction file.
When updating agent instructions, apply the same content to both files.

## Project Purpose

This repository publishes a landing page for people who want to use the HTML slide creation prompt workflow.

The page should help visitors understand:

- what the prompt set can create
- what order to use the prompts in
- what finished HTML slide outputs look like
- where to open each prompt Markdown file

This is not a presentation deck and not a portfolio page.

## Current Direction

- Use a vertical landing page, not slide-by-slide presentation sections.
- The first view uses `assets/images/hero-main.png` as the main visual.
- The hero image already contains the Japanese title text, so avoid duplicating the visible title in HTML.
- Keep an accessible `h1` for screen readers.
- Place the examples section directly after the hero.
- Prompt body text should not be embedded into the landing page.
- Prompt cards should show role, timing, and links to the Markdown files.
- The site should feel useful, clean, and landing-page-like, with slightly playful visual energy.

## Visual Assets

- `assets/images/hero-main.png`: final hero visual for the first view.
- `assets/images/character-reference.png`: character reference image for future image generation.
- Do not reuse archived slide images as active assets unless explicitly requested.
- `_archive/` contains old files and is ignored by Git.

For future generated hero or character images:

- Keep the pink-haired school-uniform character consistent with `assets/images/character-reference.png`.
- Use the character mainly in the hero or major visual moments.
- Do not force the character into every section.
- Avoid workflow diagrams, numbered steps, file names, and checklists inside hero visuals.
- Prefer bold but readable Japanese gothic/sans-serif typography when text is generated inside images.

## Examples Section

Examples are managed by `data/examples.json`.

Do not include outputs under `B:\yuki\repository\slide-archive\archive` or URLs containing `/slide-archive/archive/` in the public examples list, except these two explicitly approved legacy examples:

- `inage-election`
- `tiktok-cm`

Expected behavior:

- initial display: 3 examples
- "もっと見る": add 6 more examples at a time
- after expansion, show a close/collapse control
- if the list grows large, prefer a constrained scroll area over an endlessly long page

Example screenshots are stored in `assets/examples/screenshots/`.
They are generated by `tools/capture-first-views.js`.
The screenshot automation is an internal maintenance feature and should not be promoted as a public LP section.

## Prompt Data

Prompt cards are managed by `data/prompts.json`.

When `data/prompts.json` or `data/examples.json` changes, run:

```powershell
npm run build:data
```

This regenerates `assets/js/data.js`.

## Implementation Notes

- Keep the page static HTML/CSS/JS.
- Avoid external fonts, CDNs, frameworks, and build systems for the public page.
- Keep links relative where possible.
- Preserve `file:///` compatibility for local checks.
- Use `npm run check:data` after changing JSON data.
- Use `npm run screenshots` only when screenshots need updating.
